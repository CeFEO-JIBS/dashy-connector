Add Dashy to Claude

The connector is a remote MCP server. You add it once in Claude's settings, then enable it in any conversation. Here is the URL you will need:

MCP Server https://mcp.fbdashboard.org/mcp
Pick your plan

Pro & Max

Individual accounts
  1. Open Settings → Connectors.
  2. Click Add custom connector.
  3. Paste the MCP server URL above.
  4. Name the connector Dashy.
  5. Click Add, then sign in when prompted.

Team & Enterprise

Managed by an owner
  1. An Owner opens Organization settings → Connectors.
  2. Chooses Add custom connector.
  3. Pastes the URL and names the connector Dashy.
  4. Saves it for the organisation.
  5. Each member then connects individually and signs in.

Custom connectors are a Claude feature on Pro, Max, Team and Enterprise plans. The remote MCP server URL must use HTTPS — Dashy already does. New connectors are added from the web app or desktop app; the mobile apps can use a connector you have already added, but cannot add new ones.

Step by step

Open Connectors in settings

On Pro or Max, go to Settings → Connectors. On Team or Enterprise, an Owner does this under Organization settings → Connectors.

Add a custom connector

Click Add custom connector at the bottom of the Connectors section, then paste the Dashy MCP server URL:

https://mcp.fbdashboard.org/mcp

Name the connector Dashy so it is easy to recognise later. You can leave the transport and icon fields at their defaults, then click Add.

Sign in to authorise

Claude will open a sign-in step to authorise the connection. This grants Claude read access to the research dataset on your behalf — Claude never sees your password, and you can revoke access at any time by disconnecting the connector.

Allow Dashy's tools

Open Dashy under Settings → Connectors. In Tool permissions you will see Dashy's eight tools. Set them to Always allow so Claude can search the dataset without asking for approval on every question.

Enable it in a conversation

In a chat, use the + button near the message box, open Connectors, and switch on Dashy. Connectors are toggled per conversation, so you decide when Claude can reach the dataset.

Try a first question

Ask something the dataset can answer, for example:

Dashy, what family business research says about board
of directors? What are the main themes and the key
references?

If Claude returns the main themes with specific references, the connector is working.

Note: start your questions by addressing Dashy directly — for example, “Dashy, what do we know about…”

If something is off
The connector doesn't appear in my chat ›

Connectors are enabled per conversation. Open the + menu near the message box, choose Connectors, and toggle Dashy on. On Team and Enterprise plans, an Owner must add the connector for the organisation before members can see it.

I can't add a connector ›

Custom connectors are available on Pro, Max, Team and Enterprise. They are added from the web or desktop app, not from the mobile apps. If you are on a Team or Enterprise plan, only an Owner can add a new connector.

Sign-in failed or I need to reconnect ›

Remove the connector in Settings → Connectors and add it again with the URL above. If the sign-in step still fails, the dataset may be in a maintenance window — get in touch and we'll check.

I'm being rate-limited ›

Access is paced per user to keep capacity fair during the pilot. For larger or systematic searches, reach out and we'll talk through options.
